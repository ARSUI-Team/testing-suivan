🚨 *FINDING CRITICAL — WAJIB FIX SEBELUM SUBMIT*

1️⃣ *SEC-AC-1: AdminCap DIBIKIN tapi TIDAK DIPAKAI* (S1 CRITICAL)
Lo udah define AdminCap di arisan_pool.move:792 tapi di select_winner() dan slash_collateral() lo masih pakai pattern Solidity-style:
assert!(sender == pool.creator || sender == pool.ai_optimizer, E_NOT_OWNER);

Ini S1 karena:
- Front-running: attacker bisa liat tx di mempool, submit lebih cepat
- Relay attack: kalau creator address compromised, semua pool jatuh
- Di Sui, auth HARUS pakai Capability Object pattern

*FIX:*
public fun select_winner(
    _cap: &AdminCap,  // ← INI yang jadi auth
    pool: &mut ArisanPool,
    clock: &sui::clock::Clock,
    ctx: &mut TxContext,
) {
    // Gak usah cek sender — having AdminCap IS the auth
}

2️⃣ *SEC-AR-1: Pseudo-Random Bias* (S1 CRITICAL)
Di select_winner line 598:
let winner_index = seed % eligible_count;

seed % eligible_count itu punya integer truncation bias — beberapa index punya probabilitas lebih tinggi. Di pool kecil (5-10 orang), ini bisa di-exploit.

*FIX OPTIONS:*
- Rejection sampling (simple fix)
- Commit-reveal pattern (proper fix — pakai Seal)
- Switchboard VRF (production grade)

3️⃣ *SEC-AR-3: Precision Loss di Yield Strategy* (S2 HIGH)
Di yield_strategy.move:144:
shares_to_mint = amount * total_shares / total_deposits

Floor division = dust lost setiap deposit/withdraw. Tumpuk bisa jadi puluhan dolar.

*FIX:*
fun mul_div(a: u64, b: u64, c: u64): u64 {
    let result = (a as u128) * (b as u128) / (c as u128);
    (result as u64)
}
const MIN_SHARES_OFFSET: u64 = 1000;

🏗️ *DESIGN ISSUES — FIX SEBELUM MAINNET*

4️⃣ *DES-FN-1: public_transfer Internal = Breaks PTB*
Lo pakai transfer::public_transfer() di dalam select_winner() dan end_pool(). Ini break PTB composability — caller GAK BISA compose operation setelah payout (misal swap via DeepBook, deposit ke vault).

*FIX:* Return Coin ke caller:
public fun select_winner(...): Coin<TEST_USDC> {
    let payout_coin = coin::from_balance(payout_balance, ctx);
    payout_coin  // ← return, jangan transfer internal
}

5️⃣ *DES-OM-1: Unbounded participant_list*
participant_list: vector<address> + loop while(i < len) = O(n) gas. Kalau max_participants besar, tx bisa timeout.

*FIX:* Tambah cap:
const MAX_POOL_SIZE: u64 = 50;

6️⃣ *PAT-CP-1: Solidity-style Auth*
ctx.sender() == pool.creator itu pattern EVM. Di Sui, capability object (AdminCap, FactoryAdminCap) adalah native auth mechanism. Lo udah bikin tapi belum dipakai semuanya.
